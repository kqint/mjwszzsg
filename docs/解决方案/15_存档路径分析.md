# 存档路径与导入导出

## 存档位置

游戏使用两套 Cocos2dx 存储，均在 app 私有目录 `/data/data/com.gamedo.v360/` 下：

| 存储 | 相对路径 | 格式 | 内容 |
|:--|:--|:--|:--|
| CCUserDefault | `shared_prefs/Cocos2dxPrefsFile.xml` | SharedPreferences XML | 关卡进度、金币、物品数量、教学进度 |
| localStorage | `files/jsb.sqlite` | SQLite | 本地键值存储（Cocos2dx JS binding 使用，此游戏可能未使用） |

## 存档内容示例

从 `Cocos2dxPrefsFile.xml` 中提取的关键字段：

| 键 | 类型 | 含义 | 示例值 |
|:--|:--|:--|:--|
| `m_pGameLevelBig` | int | 大关卡 | 2 |
| `m_pGameLevelSmall` | int | 小关卡 | 7 |
| `m_pGameGold` | int | 金币 | 19 |
| `nMrdlNum` | int | 每日登录次数 | 2 |
| `nDay` / `nMonth` / `nYear` | int | 最后登录日期 | 2026-05-09 |
| `m_pTeachIndex` | int | 教学进度 | 17 |
| `enemynum0` ~ `enemynum79` | int | 各关卡杀敌数 | — |
| `Baowu0` ~ `Baowu19` | int | 宝物数量 | — |
| `Wuping0` ~ `Wuping14` | int | 物品数量 | — |
| `wujiangLevel0` ~ `wujiangLevel7` | int | 武将等级 | — |
| `wujiangState0` ~ `wujiangState7` | int | 武将状态 | — |
| `JienengLevel0` ~ `JienengLevel31` | int | 技能等级 | — |
| `maxtimetime` | int | 最长游戏时间（秒） | 290 |

注意：此存档**不含 `zhengbanFlag` 键**。由于 Z2+Z3 将默认值从 `true` 改为 `false`，新安装的游戏自动为非正版模式。

## adb restore 失败原因

`adb backup -f backup.ab -noapk com.gamedo.v360` 导出可以成功，但 `adb restore backup.ab` 恢复失败的原因：

1. **应用未声明 `android:allowBackup`**：原版 Manifest 没有此属性，默认值在不同 API 级别行为不同
2. **targetSdkVersion=22**：虽然理论上支持 backup，但实际行为不稳定
3. **OPPO/Realme 系统兼容性**：部分国产 ROM 对 `adb backup/restore` 支持不完整
4. **backup transport 问题**：设备上可能未配置合适的 backup transport

## 正确导出导入方法

### 导出存档

```bash
# 方法 1：run-as 导出到 /sdcard 再 pull（推荐）
adb shell "run-as com.gamedo.v360 cat shared_prefs/Cocos2dxPrefsFile.xml > /sdcard/Cocos2dxPrefsFile.xml"
adb pull /sdcard/Cocos2dxPrefsFile.xml

# 方法 2：直接 pull（仅适用于 API < 29 或 root）
adb pull /data/data/com.gamedo.v360/shared_prefs/Cocos2dxPrefsFile.xml

# 方法 3：备份整个 data 目录
adb shell "run-as com.gamedo.v360 tar -cf /sdcard/game_backup.tar shared_prefs files"
adb pull /sdcard/game_backup.tar

# 方法 4：adb backup（仅限导出）
adb backup -f backup.ab -noapk com.gamedo.v360
# 解压 backup.ab 需要 abe 工具：https://github.com/nelenkov/android-backup-extractor
```

### 导入存档

```bash
# 方法 1：push 到 /sdcard 再 run-as cp（推荐）
adb push Cocos2dxPrefsFile.xml /sdcard/Cocos2dxPrefsFile.xml
adb shell "run-as com.gamedo.v360 cp /sdcard/Cocos2dxPrefsFile.xml shared_prefs/Cocos2dxPrefsFile.xml"

# 方法 2：cat 重定向
adb push Cocos2dxPrefsFile.xml /sdcard/Cocos2dxPrefsFile.xml
adb shell "run-as com.gamedo.v360 cat /sdcard/Cocos2dxPrefsFile.xml > shared_prefs/Cocos2dxPrefsFile.xml"

# 方法 3：su 直接复制（需要 root）
adb push Cocos2dxPrefsFile.xml /sdcard/Cocos2dxPrefsFile.xml
adb shell "su -c 'cp /sdcard/Cocos2dxPrefsFile.xml /data/data/com.gamedo.v360/shared_prefs/Cocos2dxPrefsFile.xml'"
adb shell "su -c 'chmod 660 /data/data/com.gamedo.v360/shared_prefs/Cocos2dxPrefsFile.xml'"
adb shell "su -c 'chown u0_a666:u0_a666 /data/data/com.gamedo.v360/shared_prefs/Cocos2dxPrefsFile.xml'"
```

### 完整备份恢复流程

```bash
# === 导出 ===
# 1. 导出 SP 和 files
adb shell "run-as com.gamedo.v360 tar -cf /sdcard/game_save.tar shared_prefs files"
adb pull /sdcard/game_save.tar

# === 导入（新设备或重置后） ===
# 1. 安装游戏并运行一次（创建 data 目录结构）
# 2. 清除默认存档
adb shell "run-as com.gamedo.v360 rm -rf shared_prefs files"
# 3. 导入备份
adb push game_save.tar /sdcard/game_save.tar
adb shell "run-as com.gamedo.v360 tar -xf /sdcard/game_save.tar"
```

### 清除存档（重置游戏）

```bash
# 清除 SP 和 localStorage
adb shell "run-as com.gamedo.v360 rm -rf shared_prefs files"

# 或通过系统设置
adb shell pm clear com.gamedo.v360
```

## 为什么 /data/data/ 在文件管理器中找不到

1. **Android 10+**：`/data/data/` 目录对普通应用和文件管理器不可见
2. **SELinux 策略**：`untrusted_app` 域无法访问其他应用的 data 目录
3. **Scoped Storage**：即使 `requestLegacyExternalStorage=true`，也不影响 `/data/data/` 的访问限制
4. **唯一访问方式**：`adb shell run-as <package>` 或 root

## 存档在备份中的路径

`adb backup` 导出后，用 `abe` 工具解压 `.ab` 文件，存档结构：
```
backup/
└── apps/
    └── com.gamedo.v360/
        ├── _manifest          ← AndroidManifest 信息
        ├── sp/                ← SharedPreferences
        │   └── Cocos2dxPrefsFile.xml
        └── f/                 ← files/
            └── jsb.sqlite      ← (可能为空或不导出)
```
