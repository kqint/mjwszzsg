# 去除登录与服务器连接

## 背景

游戏启动时弹出"中国移动手机游戏"账号登录/注册窗口，登录显示"服务繁忙"。经代码分析，确认**所有后端服务器已关闭**（2014 年的游戏），所有网络请求均超时。这是造成启动黑屏 141 秒和登录窗口的根因。

## 启动流程分析

当前启动链路（以中国移动 SIM 卡为例）：

```
sanguoAndroid360.onCreate()
  ├── super.onCreate() → Cocos2dxActivity.onCreate()
  │     └── Cocos2dxHelper.init() → System.loadLibrary("game")   [快速]
  ├── getWindow().addFlags(0x80)
  └── SmsSdk.InitSmsSdk(this)
        └── SmsSdk.init(activity)
              ├── getLine1Number() → ""          [已修补，快速]
              ├── getSubscriberId() → ""         [已修补，快速]
              ├── getDeviceId() → ""             [已修补，快速]
              ├── getSimSerialNumber() → ""      [已修补，快速]
              ├── 检测运营商类型                   [本地，快速]
              ├── sendGameInfo(SingleInterfaceV2)  ← HTTP POST → 超时 30s+
              │     └── DefaultHttpClient.execute()
              │           └── connect to 121.14.211.49:8961 → 💀 死服务器
              ├── sendGameInfo(GetBillTypeByICCID) ← HTTP POST → 超时 30s+
              ├── 如果运营商=中国移动:
              │     ├── GameInterface.initializeApp()  ← CMGame SDK 初始化
              │     │     └── 弹出 "中国移动手机游戏" 登录窗口
              │     │           └── 用户点登录 → HTTP POST → gmc.g188.net → 超时
              │     │           └── 用户点注册 → HTTP POST → gmc.g188.net → 超时
              │     └── 等待登录回调 → 阻塞
              └── 其他运营商: 走对应 SDK 初始化
```

**141 秒的构成：**
- `sendGameInfo()` HTTP 超时 × 2 = 60 秒
- CMGame SDK 登录等待 + HTTP 超时 = 60+ 秒  
- ITelephony binder 超时 = 10 秒
- 其他 = 10 秒

## 解决方案

### 策略：拦截所有网络出口 + 跳过登录流程

核心思路分两层：
1. **Java 层**：修补 SmsSdk 和 CMGame SDK 的网络调用和登录入口
2. **阻止登录窗口弹出**：让 CMGame SDK 初始化快速返回，不显示 UI

### 第一阶段：修补 SmsSdk 网络调用（启动路径上的关键阻塞）

#### 修改点 1：SmsSdk.init() 中的 sendGameInfo 调用

**文件**：`smali/com/gamedo/v360/SmsSdk.smali`

**位置**：`init()` 方法中 `sendGameInfo()` 的两次调用

`sendGameInfo()` 使用 `DefaultHttpClient.execute()` 发送 HTTP POST 到 `121.14.211.49:8961`。服务器已死，每次超时 30 秒。

**修补方式**：将 `sendGameInfo()` 调用替换为直接返回空字符串 `""`，或让 `sendGameInfo()` 方法直接返回 `""`。

```
修补前: invoke-virtual {p0, vX, vY}, SmsSdk->sendGameInfo(String, String)String
        move-result-object vZ
修补后: const-string vZ, ""
```

#### 修改点 2：SmsSdk.sendGameInfo() 方法本身

**更彻底的方案**：直接修改 `sendGameInfo()` 方法，在方法开头插入 `const-string v0, ""` + `return-object v0`，使所有调用者都自动获得空响应。

这样一次性修补所有 6 处 `sendGameInfo()` 调用（包括后台线程中的 `UserBillSwitch` 和 `UserAccessNetwork`）。

### 第二阶段：跳过 CMGame SDK 登录流程

#### 修改点 3：SmsSdk.init() 中的中国移动分支

**文件**：`smali/com/gamedo/v360/SmsSdk.smali`

在 `init()` 方法中，运营商检测后有一段逻辑：

```java
if (operatorType == 1) {  // 中国移动
    // ... 设置 CMGame 相关参数 ...
    GameInterface.initializeApp(activity);  // ← 触发登录窗口!
}
```

**修补方式**：将 `GameInterface.initializeApp()` 调用替换为 NOP，跳过 CMGame SDK 初始化。

#### 修改点 4：GameInterface.initializeApp() 方法本身

**文件**：`smali/cn/cmgame/billing/api/GameInterface.smali`

**更彻底的方案**：修改 `initializeApp()` 方法，在开头直接 `return-void`。这样所有调用点都不再触发 CMGame SDK 初始化，包括：
- 登录窗口不再弹出
- 所有 `gmc.g188.net` 的网络请求不再触发
- 版本检查、推送消息、推荐游戏等全部跳过

### 第三阶段：修补后台网络调用（可选，防止后台线程持续重试）

#### 修改点 5：SmsSdk$AccessNetwork、SmsSdk$UserAccessNetwork、SmsSdk$UserBillSwitch

这三个 Runnable 在后台线程调用 `sendGameInfo()`。如果第一阶段的修改点 2 已实施，`sendGameInfo()` 直接返回空字符串，这些后台线程变为几乎空转，不会造成明显延迟。

但如果要彻底清理，可以在 Runnable 的 `run()` 方法开头直接 `return-void`。

### 第四阶段：移除 libidentifyapp.so（可选）

`libidentifyapp.so` 提供 `getAppSignature()` 等设备识别功能，被支付 SDK 使用。如果 login/支付功能已全部跳过，这个库也可以移除：

1. 从 `apktool.yml` 中确认
2. 从 `lib/armeabi/` 中删除
3. 修改所有 `System.loadLibrary("identifyapp")` 调用（如存在）

但此修改可能导致引用该库的其他代码因 `UnsatisfiedLinkError` 崩溃，需要确认所有调用点。

## 风险评估

| 修改 | 风险 | 影响 |
|:--|:--|:--|
| SmsSdk 网络调用 stub | 低 | 失去计费、用户统计功能（服务器已死，无实际影响） |
| CMGame SDK 登录跳过 | 低 | 失去中国移动计费功能，单机游戏不受影响 |
| 后台网络线程 stub | 极低 | 仅减少后台 CPU/网络消耗 |
| 移除 libidentifyapp.so | 中 | 需要确认所有 JNI 调用点 |

## 预期效果

- 启动时间从 **141 秒降至 2-3 秒**
- "中国移动手机游戏" 登录窗口**不再弹出**
- "服务繁忙" 错误**不再出现**
- 游戏直接进入 Cocos2dx OpenGL 渲染画面
- 单机玩法**完全不受影响**

## 执行计划

```
1. 修补 sendGameInfo() 方法 → 直接返回 ""
2. 修补 GameInterface.initializeApp() → 直接 return
3. 重新打包测试
4. 如果仍有延迟 → 修补 init() 中的 CMGame 分支条件
5. 清理后台线程（可选）
```
