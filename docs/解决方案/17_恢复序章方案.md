# 恢复序章（新手关卡）方案

## 问题

Z1+Z2+Z3 修补后 `zhengbanFlag` 强制为 0，`getNowMapIndex()` 跳过正版门，直接返回正常关卡索引。结果是：**序章/新手教程被跳过**，游戏直接从第一关开始。

## 根因

`getNowMapIndex()` 的正版门逻辑：

```c
int HeroShuXing::getNowMapIndex() {
    int idx = computeFromLevelData();  // 从 pNowLevelID 计算

    if (zhengbanFlag != 0) {
        // 正版模式：根据 zhengbanStep 返回特殊关卡
        if (zhengbanStep == 0)   return 39;  // ← 序章/教程
        if (zhengbanStep == 100) return 40;
        if (zhengbanStep == 200) return 41;  // ← "木牛流马" 黑屏在这里
        if (zhengbanStep == 300) return 42;
        if (zhengbanStep == 400) return 43;
        if (zhengbanStep == 500) return 44;
        return computeFallback();
    }
    return idx;  // 正常关卡（我们的 Z1 强制走这里）
}
```

- **zhengbanStep=0 → 关卡 39**：序章/教程——可正常游玩
- **zhengbanStep>0 → 关卡 40-44**：需要服务端数据 → 黑屏

Z1 将所有关卡都改为正常索引，虽然是安全的但丢掉了序章。

## 方案

### 方案 A：条件性正版门（推荐）

修改 `getNowMapIndex()`，**仅在 zhengbanStep==0 时允许正版路径**（返回序章），其他情况走正常路径：

```
当前 Z1:  无条件 B → 跳过正版门 → 始终正常索引
方案 A:   条件判断 zhengbanStep==0 → 是 → 走正版路径(返回39)
                              → 否 → 走正常路径(Z1效果)
```

**ARM 实现**（需重写 getNowMapIndex 的正版门逻辑）：

```asm
; 方案 A 伪代码
@ 0x243EE4:
  LDR  R3, zhengbanFlag_ptr
  LDR  R3, [R3]
  LDRB R3, [R3]        ; load zhengbanFlag
  CMP  R3, #0
  BEQ  normal_path     ; if flag=0, normal path
  
  LDR  R3, zhengbanStep_ptr
  LDR  R3, [R3]
  LDR  R3, [R3]        ; load zhengbanStep
  CMP  R3, #0
  BEQ  zhengban_path   ; if step=0, allow zhengban (tutorial)
  ; else: fall through
  
normal_path:
  B    loc_243F28      ; normal level index

zhengban_path:
  ; original zhengban code for level 39
  MOVS R0, #0x27       ; return level 39
  POP  {R3-R5,PC}
```

**难度**：中。需要在有限的几个字节中重写逻辑，可能需要跳转到空闲区域。

### 方案 B：直接设置 pNowLevelID（更简单）

不修改 ARM 代码。在游戏启动时通过修改存档或 JNI 回调设置 `pNowLevelID[0]` 到教程对应值。

**问题**：需要知道教程关卡的正常索引号（不是 39，而是 0-38 中的某个值）。如果教程在正常索引中不存在，此方案不可行。

### 方案 C：接受现状（最务实）

序章跳过后不影响核心玩法——第一关依然可玩。相比之前完全黑屏不可玩的状态，跳过序章是可接受的权衡。重点应该放在解决"木牛流马"黑屏上。

## 分析：教程的实际内容

从 `MapLayer::restart(int)` 的代码来看，关卡 39-44 的特殊处理只影响**相机位置和初始参数**：

```c
if (mapIndex == 42)  { pos = 100.0f; param = 5; }
elif (mapIndex == 41 || mapIndex == 44) { pos = 8000.0f; param = 1; }
else /* 39, 40, 43 */ { pos = 100.0f; param = 0; }
```

关卡的实际地图数据、敌人配置、对话脚本等应该来自**资源文件**（plist/xml），不由 zhengbanStep 控制。教程关卡只是 camera 位置不同。

**结论**：序章可以恢复且不会导致黑屏（只有 zhengbanStep=0 走正版路径）。黑屏发生在 zhengbanStep 推进到 100+ 之后。

## 建议

1. **短期**：保持现状（Z1 跳过）。序章影响不大，先确保核心玩法稳定
2. **中期**：实施方案 A——在 `getNowMapIndex()` 中做条件判断，仅 zhengbanStep==0 时放行
3. **长期**：如果"木牛流马"黑屏能解决，可以完全恢复 zhengbanFlag 机制
