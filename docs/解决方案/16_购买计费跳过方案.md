# 购买计费跳过方案

## 当前状态

B1 patch（`sendSms` 直接返回）解决了购买按钮卡死问题，但用户点击购买后物品不会到账——计价流程被跳过且没有回调通知 native 代码购买成功。

## 完整购买调用链

```
1. Native game (C++) 调用 JNI
   → sanguoAndroid360.jifei(int id)

2. jifei() 解析参数:
   bid = id / 100    (商品ID)
   sid = id % 100    (渠道ID)
   → SmsSdk.sendSms(bid, sid)

3. [原版] sendSms → Handler → billing SDK → 服务器
   [B1] sendSms → return-void (直接返回，不触发后续)

4. [原版] billing 回调 → SmsSdk.myCallBackResult(result, sendId)
   result==1 → nativePayResult(1) → SimpleGame::jifeijieguo(true, 1)  ← 成功
   result!=1 → nativePayResult(-1) → SimpleGame::jifeijieguo(false, -1) ← 失败

5. Native game 读取 sanguoAndroid360.jifei 字段获取购买的商品 ID
   → 发放对应物品
```

**问题**：B1 导致步骤 3 直接返回，步骤 4 永远不执行，native 代码不知道购买结果。

## 方案：在 jifei() 中模拟购买成功

### 修改位置

`sanguoAndroid360.jifei(int id)` 方法（`smali/com/gamedo/v360/sanguoAndroid360.smali` 行 388-458）

### 修改逻辑

在 `sendSms()` 调用后，return 之前，插入：

```java
// 原代码:
sendSms(bid, sid);  // B1 已 stub → 立即返回
return;

// 修改后:
sendSms(bid, sid);  // B1 已 stub → 立即返回
jifeiBack(id);       // 存储购买项 ID 到 jifei 字段
nativePayResult(1);  // 通知 native 购买成功
return;
```

### Smali 修改

在 `sendSms` 调用（行 454）和 `return-void`（行 457）之间插入 2 行：

```smali
# 454: invoke-virtual {v0, v1, v2}, SmsSdk->sendSms(II)V
# [insert here]
# 457: return-void

# 插入:
invoke-static {p0}, Lcom/gamedo/v360/sanguoAndroid360;->jifeiBack(I)V
const/4 v0, 0x1
invoke-static {v0}, Lcom/gamedo/v360/sanguoAndroid360;->nativePayResult(I)V
```

### 数据流

```
用户点击购买(参数 id=XXX)
  → jifei(XXX)
    → jifeiBack(XXX)         ← jifei 字段 = XXX，native 读到此 ID
    → nativePayResult(1)     ← 通知 native "购买成功"
      → SimpleGame::jifeijieguo(true, 1)
        → 根据 jifei 字段(XXX) 查找对应物品
        → 发放物品到背包
```

## 风险评估

| 风险 | 说明 | 缓解 |
|:--|:--|:--|
| 所有商品免费 | 任何购买按钮都直接成功，无需付费 | 单机游戏无影响 |
| 商品 ID 映射错误 | 不同商品需要不同的 id | jifeiBack 保存原始 id，native 自行解析 |
| 重复购买 | 每次点击都会发放物品 | 可接受（单机游戏） |

## 备选方案

如果直接调用 `nativePayResult(1)` 不够（native 代码可能还检查其他状态），可以考虑：

1. **完整模拟 myCallBackResult 流程**：在 jifei 中也调用 `myCallBackResult(1, XXX)`
2. **直接操作 sentNotification flag**：`_isSend = false` 确保不会重复发送
3. **清除游戏存档后测试**：确保没有之前的失败状态残留
