# 方案D：装箱方案（兜底）

## 适用场景

方案A（降低 targetSdkVersion）失败，且方案B（全量修补）太费时，或存在无法修复的原生层兼容性问题。

## 可用方案

### 1. VMOS Pro 虚拟机

- 在 Android 11 设备上运行一个 Android 5.1/7.1 的虚拟系统
- 虚拟机内的系统对老 APK 完全兼容
- 直接安装原版 APK，无需任何修改
- 免费版足够使用

### 2. 云手机

- 红手指、多多云等
- 完全不受本地设备 Android 版本影响
- 适合挂机类游戏
- 需要网络连接

### 3. Android 模拟器（PC 端）

- 使用 Android 5.1 ~ 7.0 的 AVD
- 完全兼容老 APK
- 适合测试和开发验证

## 对比

| | VMOS Pro | 云手机 | 模拟器 |
|:--|:--|:--|:--|
| 在手机上运行 | ✅ | 需网络 | ❌ PC端 |
| 免费 | 基础版免费 | 有免费额度 | ✅ |
| 性能 | 一般（虚拟化损耗） | 依赖网络 | 取决于PC |
| 适合场景 | 日常玩 | 挂机 | 开发测试 |
